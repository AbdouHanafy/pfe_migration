Summary of work completed (2026-02-06)

1) Frontend integration
- Static frontend (frontend/index.html, frontend/app.js) now supports API key input and sends X-API-Key.
- Added Auth card (register/login/logout) to static UI. JWT token is stored in localStorage and sent automatically.

2) React frontend structure refactor (frontend/frontend-app/src)
- Best-practice structure applied:
  - pages/: HomePage.jsx, LoginPage.jsx, RegisterPage.jsx, AboutPage.jsx, NotFoundPage.jsx
  - store/: AuthContext.jsx
  - hooks/: useAuth.js, useLogger.js
  - services/: api.js, authService.js
  - utils/: validators.js, time.js
  - routes/: index.jsx (route config)
  - pages/index.js (barrel)
- App.jsx simplified to map routes from routes/index.jsx.
- Error handling improved in authService: shows user-friendly Pydantic errors.

3) Backend auth fix
- Fixed a bug in src/api/main.py: API key/JWT validation moved into _require_auth
  (it was incorrectly placed inside _create_token).

4) VMware Workstation discovery support
- Added src/discovery/vmware_workstation_discoverer.py
  - Scans .vmx files in VMWARE_WORKSTATION_PATHS
  - Parses displayName, disks (.vmdk), basic specs
- Added config in src/config.py:
  - VMWARE_WORKSTATION_PATHS (comma-separated, default ~/vmware)
- Added API endpoints in src/api/main.py:
  - GET /api/v1/discovery/vmware-workstation
  - GET /api/v1/discovery/vmware-workstation/{vm_name}
- UI (React HomePage) now lets user choose discovery source (KVM or VMware Workstation).

5) Analyze/Plan/Start for VMware Workstation
- Added source parameter to migration endpoints:
  - POST /api/v1/migration/analyze/{vm_name}?source=...
  - POST /api/v1/migration/plan/{vm_name}?source=...
  - POST /api/v1/migration/start/{vm_name}?source=...
- Frontend sends selected source automatically after discovery.

6) Dependencies
- Installed libvirt-python to enable KVM discovery.
- Pinned bcrypt to <4 in requirements.txt to avoid passlib crash.

- Simulated migration with status/reporting
- Basic auth UI (JWT) and API key support
